# Edge Bootstrap — Raspberry Pi 5 Setup

## Hardware layout

| Device | Role |
|--------|------|
| SD card (`/dev/mmcblk0p2`) | OS only — do not write project data here |
| SSD (`/dev/nvme0n1p3`, `/mnt/ssd`) | All project files, Docker images, models, databases |

## Prerequisites

- Raspberry Pi 5 with Hailo-8 HAT installed
- Raspberry Pi OS 64-bit (bookworm)
- NVMe SSD pre-formatted as ext4, mounted (or will be mounted) at `/mnt/ssd`
- Internet connection

---

## Step 1 — Run the bootstrap script (once, on the RPi)

```bash
sudo bash scripts/edge-bootstrap.sh
```

The script is **idempotent** — safe to re-run after failures or reboots.

**What it does:**
1. Checks ARM64 Linux + root
2. Detects the Hailo-8 HAT (`/dev/hailo0` or `hailortcli scan`)
3. Installs Docker + Compose plugin via the official convenience script
4. Reminds you to install HailoRT if not found (manual — see below)
5. Ensures `/dev/nvme0n1p3` is mounted at `/mnt/ssd`, adds UUID entry to `/etc/fstab`
6. Creates project directory tree: `data/db`, `data/mqtt`, `models/hailo`, `models/whisper`, `logs`, `certs`
7. Creates convenience symlink `/opt/iot-hub → /mnt/ssd/iot-hub`
8. Moves Docker data root from SD to `/mnt/ssd/docker` (prevents SD card from filling up)
9. Adds the calling user to the `docker` group

After the script:

```bash
newgrp docker   # activate docker group without re-login
```

---

## Step 2 — Clone the repo to SSD

```bash
git clone git@github.com:your-org/iot-hub.git /mnt/ssd/iot-hub/repo
cd /mnt/ssd/iot-hub/repo
```

---

## Step 3 — Configure environment

```bash
cp .env.example .env
nano .env
```

Required secrets (the compose file will refuse to start without them):

| Variable | Description |
|----------|-------------|
| `POSTGRES_PASSWORD` | Strong password for PostgreSQL |
| `GRAFANA_ADMIN_PASSWORD` | Grafana admin password |
| `RTSP_URL` | Camera stream URL (e.g. `rtsp://192.168.1.x:554/stream`) |

---

## Step 4 — Generate MQTT TLS certificates

Port 8883 requires mTLS. Run the cert generator once:

```bash
bash scripts/gen-mqtt-certs.sh
```

This places `ca.crt`, `broker.crt`, `broker.key` into `hub/mosquitto/certs/`.
Client certs (for ESP32, VPS bridge) are generated by the same script.

---

## Step 5 — Install auto-updater (pulls from GitHub every 5 min)

```bash
sudo bash scripts/install-updater.sh
```

The script installs a `systemd` timer that:
1. Runs `git fetch` every 5 minutes
2. If new commits exist on `main` — runs `git pull`, rebuilds changed images, `docker compose up -d`
3. Logs everything to `/mnt/ssd/iot-hub/logs/updater.log`

**Prerequisite:** the RPi must have SSH access to GitHub:

```bash
# Generate a key if needed
ssh-keygen -t ed25519 -C "iot-hub-rpi"

# Copy the public key and add it at github.com/settings/keys
cat ~/.ssh/id_ed25519.pub
```

Useful commands after install:

```bash
# Check timer schedule
systemctl status iot-hub-updater.timer

# Trigger an update manually right now
systemctl start iot-hub-updater.service

# Watch update log
tail -f /mnt/ssd/iot-hub/logs/updater.log
```

---

## Step 6 — Start services

```bash
make up-edge
```

Or directly:

```bash
docker compose --env-file .env -f hub/docker-compose.edge.yml up -d
```

To watch logs:

```bash
make logs-edge
# or
docker compose -f hub/docker-compose.edge.yml logs -f
```

To check status:

```bash
make ps-edge
```

---

## Step 7 — HailoRT manual install

The bootstrap script cannot auto-download HailoRT — it requires a hailo.ai developer account.

1. Go to https://hailo.ai/developer-zone/
2. Download the `.deb` package for your OS (bookworm arm64)
3. `sudo dpkg -i hailort_*.deb`
4. Reboot and re-run the bootstrap script

## Useful locations on the RPi

```
/mnt/ssd/iot-hub/
├── repo/           ← git clone here
├── data/
│   ├── db/         ← PostgreSQL bind mount (if used)
│   └── mqtt/       ← Mosquitto persistence backup
├── models/
│   ├── hailo/      ← .hef model files
│   └── whisper/    ← Whisper GGUF/ONNX files
├── logs/
├── certs/          ← extra TLS certs / CA
└── docker/         ← Docker data root (images, volumes)

/opt/iot-hub        ← symlink → /mnt/ssd/iot-hub
```

Named Docker volumes (`pgdata`, `mosquitto-data`, etc.) live in `/mnt/ssd/docker/volumes/`.
